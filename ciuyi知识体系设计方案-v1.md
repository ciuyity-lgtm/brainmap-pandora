# ciuyi 知识体系设计方案 v1.0

> 基于 Karpathy LLM Wiki + Obsidian MCP + Hermes Agent 的贾维斯大脑架构
> 参考：Karpathy autoresearch、Karpathy LLM Wiki、ACCESS 知识管理、MCP-Obsidian

---

## 一、核心理念

### 知识传导 = 递归升级

```
来源（原始材料）
   ↓ Agent 阅读 + 萃取
知识（原子笔记/双链互联）
   ↓ Agent 执行任务时使用
项目（活的项目/状态/决策）
   ↓ Agent 复盘 + 萃取
教训库（踩坑记录/可复用模式）
   ↑ Agent 下次自动防坑
```

**Karpathy 三文件夹结构的启发：**
Karpathy 的模式极其简单但有效——3个文件夹 + Agent 自动维护。我在此基础上扩展为适合 ciuyi 多项目生态的结构。

### 与传统 PARA 的区别
| 维度 | PARA | 本方案 |
|:----|:-----|:-------|
| 核心 | 项目驱动 | **知识传导驱动** |
| Agent 角色 | 用户操作 | **Agent 自动维护** |
| 递归升级 | 无 | **每次任务→萃取→下次利用** |
| 知识来源 | 用户主动收集 | **Agent 从代码/对话/项目中萃取** |

## 三、递归升级流程

### 单次任务循环

```
任务下达
    ↓
1. 加载上下文（Obsidian ← 20-知识/ + 30-项目/ + 51-教训库/）
    ↓
2. 执行任务（代码/写作/分析/决策）
    ↓
3. 萃取关键知识
    → 新发现 → 写入 20-知识/
    → 项目变更 → 更新 30-项目/状态日志
    → 踩坑 → 写入 51-教训库/
    ↓
4. 更新工作日志（21-ciuyi-hermes/工作日志）
    ↓
5. 下次同类任务 → 从步骤1开始
        ↑  知识越来越厚，Agent越来越聪明
```

### 周度/月度升级

```
每周/月：
→ Agent 遍历 51-教训库/，归纳高频模式
→ 更新 20-知识/ 中对应的方法论
→ 优化 Agent 自身行为规范（CIUYI.md）
→ 产出升级报告
```

## 四、技术链路

```
Hermes Agent (ciuyi-hermes)
    │
    ├── MCP: obsidian-mcp ←→ Obsidian vault（直接读写笔记）
    │   └── create-note / search-vault / edit-note / add-tags
    │
    ├── 文件系统: read_file/write_file/patch（大文件操作）
    │
    ├── 终端: terminal（代码执行/数据分析）
    │
    └── 记忆系统: memory（跨会话持久记忆）
```

**obsidian-mcp 提供的 11 个工具：**
- `create-note` / `edit-note` / `read-note` / `delete-note`
- `search-vault`（FTS5 全文搜索）
- `add-tags` / `remove-tags` / `rename-tag`
- `create-directory` / `move-note` / `list-available-vaults`
