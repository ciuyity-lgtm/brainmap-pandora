# ciuyi 新知识体系设计方案

> 基于 Karpathy Loop + Karpathy LLM Wiki + Obsidian MCP 的三位一体架构
> 2026-06-15 ciuyi-hermes

---

## 一、核心理念

### 三条原则

1. **Agent 写，人读** — 知识库不是人手动整理的，是 Agent 在项目中自动萃取的
2. **递归升级** — 每次任务后自动萃取→写入→下次调用→持续进化
3. **双链即传导** — 笔记之间的 [[链接]] 就是知识传导的物理路径

### 架构对标

| 你的现在 | 未来 |
|:---------|:-----|
| 道法术器（固定分类） | ACCESS（动态生长） |
| Obsidian 手动维护 | Agent 自动写入+健康检查 |
| 知识断裂 | MCP 实时双向连接 |
| CherryStudio 读不到 | Hermes/Obsidian 实时同步 |

### 各目录详细说明

#### 00-总控/
| 文件 | 说明 | 谁维护 |

#### 10-来源/
- 原始材料，不修改
- 命名：`作者-标题-日期.md`
- 标签：`#来源` `#状态/待处理` `#状态/已萃取`
- Agent 处理完来源后将标签改为 `#状态/已萃取`

#### 20-知识/（核心——原子笔记）
- **一篇笔记一个概念**，Karpathy 模式
- 格式：
```markdown
---
tags: [领域, 概念]
来源:: [[10-来源/xxx]]
---

# 概念名

核心要义：一句话说清楚

## 详细内容
...

## 相关概念
- [[链接1]]
- [[链接2]]
```
- Agent 自动从 `10-来源/` 中萃取生成
- 自动建立双链关联

## 三、递归升级流程（Karpathy Loop 简化版）

```
┌─────────────────────────────────────────────────┐
│  ciuyi-hermes 任务循环                            │
│                                                   │
│  1. 接收任务（老板说"做XX"）                       │
│  2. 自动加载相关笔记（30-项目/ + 20-知识/）        │
│  3. 执行任务（代码/分析/写报告）                    │
│  4. 萃取关键知识                                   │
│     ├─ 写 20-知识/（新概念）                        │
│     ├─ 更新 30-项目/（状态+决策）                   │
│     └─ 写 50-递归升级/教训库（踩坑记录）            │
│  5. 输出结果给老板                                  │
│  6. 回到 1（下次任务自动加载 4 写入的新知识）        │
└─────────────────────────────────────────────────┘
```

每一轮循环，知识库都比上一轮更丰富 → Agent 的上下文更精准 → 产出质量更高。
